Fun with Fonts: Rendering and Interpolating Fonts

Team Members:

Austin Patel, Ashley Chang, Dominic De Bettencourt, Jennessa Ma

Project Summary:

This project focuses on rendering fonts given a TrueType file and writing a Bezier curve rasterizer to display these fonts. This involves being able to display the font outline and filling the characters. We also will create a font interpolator: a program that takes in two font files and creates intermediate fonts that are a weighted blend of the fonts by interpolating the Bezier curves in the fonts.

Problem Description:

The problem we are aiming to solve in this project is rendering fonts given TrueType font files.  We will load the font and then perform a rasterization process to display characters from the font. This involves using the CPU to compute and render the bezier curves and antialiasing as well as filling in the boundaries defined by the curves.

After creating a basic font renderer, we will perform font interpolation, which is the process of interpolating between two different fonts. This will allow us to generate intermediary fonts that are blends of existing fonts as well as create an animation showing the progression of one font to another.

Although font rendering is a very commonly implemented feature, our additional font interpolating extension will allow us to render “blends” of fonts as well (and potentially export them to TrueType files). The interpolation procedure is quite complex and involves finding a contour mapping between two completely different fonts, possibly needing the reduction method to allow lerping of bezier curves, and then interpolating the final mapping of bezier curves.

At a high level, we will accomplish this project by loading in font files using an existing library, implementing the CPU rendering and font interpolation in C++.

Stretch goal (time permitting):

As a reach goal if we have additional time, we hope to move the font rendering process onto the GPU rather than the CPU. Rendering fonts using the GPU involves drawing the font using a series of triangles rather than the boundary tests involved with the CPU approach. Rendering with triangles lets us render in parallel using GPU resources, which provides a speed improvement over CPU rendering.

Goals and Deliverables:

Main Deliverables:

  1. Read in TrueType font file and parse FontStream data
  1. Use Aspose TrueType Font C++ library
  1. Generate quadratic bezier curves to draw font outlines
  1. Load Bezier control points directly from a TrueType font file
  2. Use the Bezier approach we learned in class to determine an equation for the Bezier curve (de Casteljau algorithm).
  3. Rasterize the Bezier curves (follow Bezier curve computation from project 2 and follow rasterization approach from project 1)
  4. Create visualizations of fonts rendered with just the font outline
  1. Filling in the fonts using boundary tests
  1. The Bezier curves define the boundaries of the font and we will use a boundary test to fill in each pixel whether or not it is within the font (similar to the approach for rasterizing triangles in project 1 where we iterate through every pixel in the grid).
  2. Output images of font without anti-aliasing (requires computing how we wish to scale the Bezier curves such that the font fits within the region we are rasterizing). Also requires computing spacing between glyphs so that we can render words.
  3. Create visualizations showing words rendered across a number of different fonts.
  1. Anti-alias with supersampling
  1. Output smoother images using supersampling as described in Lecture 3
  2. Create visualizations of font renders at varying sample rates
  1. Interpolate between different TrueType fonts
  1. Create an algorithm to map glyphs to each other using the Contour-Naivete and the Reduction Method for glyphs that have a greater number of contours than the other (see FontLerp paper which we reference in the resources section). This will give us correspondences between Bezier curves in each of the fonts.
  2. Once we know the matching components, interpolating between the two involves lerping between the control points, which will change the resulting Bezier curves.
  3. Create visualizations of font interpolation using various t values (gifs/videos) which will show how one font transforms to another (showing both success and failure cases).

Evaluation metrics:

  1. We will know that we are rendering fonts correctly and with high quality by comparing the rendering of well known fonts between our renderer and the rendering in Google Docs.
  2. Evaluating the intermediary fonts will be more subjective in nature.

Stretch goal deliverables:

  1. (Stretch goal) Implement GPU font rendering
  1. Setup GPU based rendering (MacOS has Metal has a C++ interface and there are also other cross-platform rendering approaches if we have time to pursue this path)
  2. Drawing the font using a series of triangles rather than the boundary tests involved with the CPU approach (the article we link for GPU font rendering involves a pixel flipping approach using triangles on the GPU).
  3. Create visualizations comparing CPU font rendering vs. GPU font rendering as well as performance comparisons
